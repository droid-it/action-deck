# Phase 1: Project Skeleton — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Lay down the compilable Swift Package skeleton with four targets, a Hummingbird 2 server that responds to `GET /ping`, a seeded default config file, and a Svelte 5 client scaffold. No business logic yet — just proof that every piece of the stack starts.

**Architecture:** Monorepo with one Swift Package at the root (four targets: `ActionDeckCore` library, three executables for Server/App/MCP) and a sibling `client/` directory for the Svelte 5 web UI. Executables are implemented with Swift's `@main` attribute so we can use `async` consistently starting at Task 3.

**Tech Stack:**
- Swift 6.0, strict concurrency
- Swift Package Manager (no Xcode project)
- [Hummingbird 2](https://github.com/hummingbird-project/hummingbird) (async-native HTTP server)
- macOS 14 Sonoma minimum (for `MenuBarExtra` later, and modern `Observation`)
- Svelte 5 + TypeScript + Vite for the web client
- Node.js 20+ for tooling

**Swift quick-reference** (for Kotlin folks):
- `let` = `val` (immutable), `var` = `var`
- `struct` = `data class` (value type). `class` = `class` (reference type)
- `enum` with associated values = `sealed class` hierarchy
- `@main` on a type with `static func main() async throws` replaces Kotlin's `fun main()`
- `Package.swift` is like `build.gradle.kts` — declarative, Swift-as-DSL

- [ ] **Step 2.1: Write `Package.swift`**

Create `/Users/apple/PersonalProjects/action-deck/Package.swift`:

```swift
// swift-tools-version: 6.0
// ^^^ first line is mandatory — tells SPM which toolchain features are available.

import PackageDescription

let package = Package(
    name: "ActionDeck",

    // macOS 14 = Sonoma; required for MenuBarExtra and modern Observation.
    platforms: [.macOS(.v14)],

    // Products are what consumers import. Libraries are linked into executables;
    // executables are built as binaries under .build/debug/.
    products: [
        .library(name: "ActionDeckCore", targets: ["ActionDeckCore"]),
        .executable(name: "ActionDeckServer", targets: ["ActionDeckServer"]),
        .executable(name: "ActionDeckApp", targets: ["ActionDeckApp"]),
        .executable(name: "ActionDeckMCP", targets: ["ActionDeckMCP"]),
    ],

    // External dependencies. Hummingbird is added in Task 4.
    dependencies: [],

    // Targets map one-to-one with Sources/<name>/ directories by default.
    targets: [
        .target(
            name: "ActionDeckCore"
        ),
        .executableTarget(
            name: "ActionDeckServer",
            dependencies: ["ActionDeckCore"]
        ),
        .executableTarget(
            name: "ActionDeckApp",
            dependencies: ["ActionDeckCore"]
        ),
        .executableTarget(
            name: "ActionDeckMCP",
            dependencies: ["ActionDeckCore"]
        ),
    ]
)
```

- [ ] **Step 4.3: Implement `/ping` route**

Replace `Sources/ActionDeckServer/App.swift` with:

```swift
import Hummingbird

// Hummingbird 2 builds the app from a Router plus an Application wrapper.
// RequestContext (generic) carries per-request state; BasicRequestContext
// is the out-of-the-box implementation with logger + decoder/encoder.
@main
struct ActionDeckServerApp {
    static func main() async throws {
        // Router holds the route table. It is generic over the context type
        // so you can swap BasicRequestContext for a custom one later.
        let router = Router()

        // Closures handle requests. The return value is encoded via the
        // context's ResponseEncoder. Returning a String writes text/plain.
        router.get("/ping") { _, _ in
            "pong"
        }

        // Bind to localhost only. 9900 is the chosen port (see AppConfig.settings.port).
        let app = Application(
            router: router,
            configuration: .init(
                address: .hostname("127.0.0.1", port: 9900),
                serverName: "ActionDeck/0.1"
            )
        )

        print("ActionDeckServer listening on http://127.0.0.1:9900")
        try await app.runService()
    }
}
```

## Red flags while executing

- **Hummingbird resolution fails** → check Swift version (`swift --version`); needs 6.0. Older Swift will not accept the `swift-tools-version: 6.0` header on line 1 of `Package.swift`.
- **`swift build` hangs on "Fetching"** → proxy/network. `swift package resolve` gives clearer errors.
- **Vite scaffold creates a non-Svelte-5 project** → confirm template flag is `svelte-ts` (not `svelte` which can be v4 depending on Vite version). Output of `npm ls svelte` is ground truth.
- **Port 9900 already in use** → `lsof -iTCP:9900 -sTCP:LISTEN` to find the culprit. Do not change the port; it is the documented default.
